# Segurança da versão pública

Este repositório deve ser mantido como uma versão de portfólio/código limpo.

## Não publicar

- Dados reais de pessoas
- Bancos locais reais
- Contracheques
- Boletins internos
- PDFs administrativos reais
- Planilhas sensíveis
- Modelos oficiais internos preenchidos
- Senhas, tokens ou credenciais
- Caminhos locais de máquina
- Ambientes virtuais
- Pastas de build/publicação

## Antes de cada commit

1. Rodar `git status`.
2. Conferir arquivos novos.
